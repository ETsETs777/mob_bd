## Фаза 1 — Стабильность и UX (1–2 недели)

### P0 — Надёжность данных

- [x] **Единый слой обработки ошибок** — централизованный `ApiException`, retry с exponential backoff (3 попытки)
- [x] **Graceful degradation** — если CoinGecko недоступен, показывать MOEX + валюты без блокировки всего экрана
- [x] **Индикатор источника данных** — на каждой карточке badge «MOEX» / «Frankfurter» / «кэш»
- [x] **Время последнего обновления** — на всех экранах, не только на главной
- [x] **Pull-to-refresh** — унифицировать и обновлять `lastRefreshTime` на каждом экране

### P0 — Конфигурация

- [x] **Файл `.env.example`** — шаблон для `COINGECKO_KEY` и `FINNHUB_KEY`
- [x] **Экран настроек** — ввод API-ключей без пересборки (Hive, локально)
- [x] **Выбор базовой валюты** — USD / RUB / EUR для отображения всех котировок

### P1 — UI/UX

- [x] **Поиск по активам** — на экране «Рынки» (крипто + акции)
- [x] **Избранное (watchlist)** — локальное хранение 5–10 тикеров, отдельная вкладка или секция на главной
- [x] **Переключатель периода графика** — 7d / 30d / 1y на экране деталей
- [x] **Hero-анимация** — переход с карточки на экран деталей
- [x] **Haptic feedback** — при pull-to-refresh и смене вкладок
- [x] **Локализация RU/EN** — settings + onboarding (база была ранее)

### P1 — Исправления из v1.0

- [x] Конвертер валют — фикс `TextEditingController` (не пересоздавать на каждый build)
- [x] Прогрессивная загрузка главной — секции грузятся независимо
